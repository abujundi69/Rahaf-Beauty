RAHAF BEAUTY - BACKEND DATABASE AND API SPECIFICATION
=====================================================

1. PROJECT OVERVIEW
-------------------

System name:
RAHAF BEAUTY

System type:
Arabic-first ecommerce beauty/cosmetics system for browsing products, managing cart/wishlist, placing cash-on-delivery orders, and managing the store through an admin dashboard.

Frontend behavior to preserve in the backend:
- Arabic-only storefront and admin UI.
- RTL by default.
- English digits in all formatted numbers.
- Frontend roles:
  - Admin: store management only.
  - Customer: shopping, cart, wishlist, orders, profile, addresses, account deletion.

Backend roles:
- Admin
- Customer

Recommended backend style:
- REST API.
- JSON request/response bodies.
- Token or secure cookie based authentication.
- Use server-side authorization checks for every admin/customer endpoint.
- Store dates in UTC.
- Keep order snapshots immutable enough for historical records.


2. DATABASE TABLES
------------------

Table: Users
Purpose:
Stores login identity, role, account deletion state, and core authentication data.

Fields:
- Id: uuid, required, primary key.
- FullName: nvarchar(200), required for active customers/admins.
- Email: nvarchar(320), required for active users, nullable/anonymized after deletion.
- PhoneNumber: nvarchar(40), required for active customers, nullable/anonymized after deletion.
- PasswordHash: nvarchar(500), required.
- Role: enum UserRole, required.
- IsDeleted: bit/boolean, required, default false.
- DeletedAt: datetime nullable.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Unique filtered index on Email where IsDeleted = false.
- Unique filtered index on PhoneNumber where IsDeleted = false, if phone login or uniqueness is required.
- Index on Role.
- Index on IsDeleted.

Relationships:
- One User has one CustomerProfile.
- One User has many CustomerAddresses.
- One User has one Cart.
- One User has one Wishlist.
- One User has many Orders.
- One User has many ProductReviews.

Notes:
- Deleted accounts must not be able to login.
- Email and phone should be removed or anonymized during customer account deletion.
- Do not physically delete Users if audit trails are needed; use IsDeleted.


Table: CustomerProfiles
Purpose:
Stores customer-specific profile data separate from authentication.

Fields:
- Id: uuid, required, primary key.
- UserId: uuid, required, foreign key to Users.Id.
- PreferredName: nvarchar(150), optional.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Unique index on UserId.

Relationships:
- Belongs to Users.

Notes:
- Delete or anonymize this row when a customer deletes the account.


Table: CustomerAddresses
Purpose:
Stores saved delivery addresses for customers.

Fields:
- Id: uuid, required, primary key.
- UserId: uuid, required, foreign key to Users.Id.
- City: nvarchar(120), required.
- Area: nvarchar(120), required.
- Street: nvarchar(200), required.
- Building: nvarchar(100), optional.
- Notes: nvarchar(1000), optional.
- IsDefault: bit/boolean, required, default false.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Index on UserId.
- Filtered index on UserId + IsDefault where IsDefault = true.

Relationships:
- Belongs to Users.

Notes:
- Remove all addresses when the customer deletes the account.
- Orders store their own address snapshot, so deleting an address must not affect existing orders.


Table: Brands
Purpose:
Stores product brands used for filtering, products, and brand-level discounts.

Fields:
- Id: uuid, required, primary key.
- Name: nvarchar(200), required.
- Slug: nvarchar(220), required.
- Description: nvarchar(max), optional.
- IsActive: bit/boolean, required, default true.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Unique index on Slug.
- Index on IsActive.

Relationships:
- One Brand has many Products.
- One Brand can have many Discounts where Type = Brand.

Notes:
- Visible names in the UI should be Arabic.
- Slugs can be internal and do not need to be shown to customers.


Table: Categories
Purpose:
Stores product categories for navigation, filtering, and product grouping.

Fields:
- Id: uuid, required, primary key.
- Name: nvarchar(200), required.
- Slug: nvarchar(220), required.
- Description: nvarchar(max), optional.
- ImageUrl: nvarchar(1000), optional.
- IsActive: bit/boolean, required, default true.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Unique index on Slug.
- Index on IsActive.

Relationships:
- One Category has many Products.

Notes:
- Category names and descriptions should be Arabic for the frontend.


Table: Products
Purpose:
Stores core product data and base pricing.

Fields:
- Id: uuid, required, primary key.
- Name: nvarchar(250), required.
- Slug: nvarchar(280), required.
- BrandId: uuid, required, foreign key to Brands.Id.
- CategoryId: uuid, required, foreign key to Categories.Id.
- BasePrice: decimal(18,2), required.
- BaseOldPrice: decimal(18,2), optional.
- Description: nvarchar(max), optional.
- Ingredients: nvarchar(max), optional.
- HowToUse: nvarchar(max), optional.
- SkinType: nvarchar(200), optional.
- IsActive: bit/boolean, required, default true.
- IsNew: bit/boolean, required, default false.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Unique index on Slug.
- Index on BrandId.
- Index on CategoryId.
- Index on IsActive.
- Index on IsNew.
- Full-text/search index on Name and Description if supported.

Relationships:
- Belongs to Brand.
- Belongs to Category.
- Has many ProductImages.
- Has many ProductVideos.
- Has many ProductColors.
- Has many ProductSizes.
- Has many ProductVariants.
- Has many ProductReviews.
- Has many CartItems.
- Has many OrderItems through ProductId, nullable for historical preservation.

Notes:
- Admin must not manually set Rating or ReviewCount.
- Rating is calculated from ProductReviews.
- BasePrice is used when no selected size/variant overrides price.


Table: ProductImages
Purpose:
Stores product image media.

Fields:
- Id: uuid, required, primary key.
- ProductId: uuid, required, foreign key to Products.Id.
- ImageUrl: nvarchar(1000), required.
- AltText: nvarchar(250), optional.
- SortOrder: int, required, default 0.
- CreatedAt: datetime required.

Indexes:
- Index on ProductId.
- Index on ProductId + SortOrder.

Relationships:
- Belongs to Products.


Table: ProductVideos
Purpose:
Stores product video media.

Fields:
- Id: uuid, required, primary key.
- ProductId: uuid, required, foreign key to Products.Id.
- VideoUrl: nvarchar(1000), required.
- SortOrder: int, required, default 0.
- CreatedAt: datetime required.

Indexes:
- Index on ProductId.
- Index on ProductId + SortOrder.

Relationships:
- Belongs to Products.


Table: ProductColors
Purpose:
Stores product color options.

Fields:
- Id: uuid, required, primary key.
- ProductId: uuid, required, foreign key to Products.Id.
- Name: nvarchar(120), required.
- HexCode: nvarchar(20), required.
- CreatedAt: datetime required.

Indexes:
- Index on ProductId.

Relationships:
- Belongs to Products.
- Can be referenced by ProductVariants, CartItems, and OrderItems snapshots.

Notes:
- HexCode must be a valid color such as #E9B0BF.


Table: ProductSizes
Purpose:
Stores product size options with size-specific pricing and stock.

Fields:
- Id: uuid, required, primary key.
- ProductId: uuid, required, foreign key to Products.Id.
- Label: nvarchar(80), required.
- Price: decimal(18,2), optional. If null, use Products.BasePrice.
- OldPrice: decimal(18,2), optional.
- Stock: int, optional. If null, use product-level stock or variant stock.
- SKU: nvarchar(120), optional.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Index on ProductId.
- Unique index on ProductId + Label if duplicate size labels are not allowed.
- Unique filtered index on SKU where SKU is not null.

Relationships:
- Belongs to Products.
- Can be referenced by ProductVariants, CartItems, and OrderItems snapshots.

Notes:
- Size-specific price must be used in product detail, cart, checkout, and order totals.
- Stock cannot be negative.


Table: ProductVariants
Purpose:
Stores optional concrete combinations of product color and size with price, stock, and SKU.

Fields:
- Id: uuid, required, primary key.
- ProductId: uuid, required, foreign key to Products.Id.
- ProductColorId: uuid, optional, foreign key to ProductColors.Id.
- ProductSizeId: uuid, optional, foreign key to ProductSizes.Id.
- Price: decimal(18,2), optional.
- OldPrice: decimal(18,2), optional.
- Stock: int, required, default 0.
- SKU: nvarchar(120), optional.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Index on ProductId.
- Index on ProductColorId.
- Index on ProductSizeId.
- Unique index on ProductId + ProductColorId + ProductSizeId where needed.
- Unique filtered index on SKU where SKU is not null.

Relationships:
- Belongs to Products.
- Optionally belongs to ProductColors.
- Optionally belongs to ProductSizes.

Notes:
- Use this table if color-size stock/pricing combinations are required.
- If not using concrete variants, ProductSizes and ProductColors are enough.

Stores one wishlist per customer.

Fields:
- Id: uuid, required, primary key.

Indexes:
- Unique index on UserId.

Relationships:

Purpose:
Stores products saved by a customer.

Fields:
- Id: uuid, required, primary key.

- ProductId: uuid, required, foreign key to Products.Id.

Indexes:
- Index on WishlistId.

Relationships:

Table: Carts
Purpose:
Stores one active cart per customer.

Fields:
- Id: uuid, required, primary key.
- UserId: uuid, required, foreign key to Users.Id.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Unique index on UserId.

Relationships:
- Belongs to Users.
- Has many CartItems.


Table: CartItems
Purpose:
Stores selected cart lines, including selected variant and price snapshot at add/update time.

Fields:
- Id: uuid, required, primary key.
- CartId: uuid, required, foreign key to Carts.Id.
- ProductId: uuid, required, foreign key to Products.Id.
- ProductSizeId: uuid, optional, foreign key to ProductSizes.Id.
- ProductColorId: uuid, optional, foreign key to ProductColors.Id.
- ProductVariantId: uuid, optional, foreign key to ProductVariants.Id.
- Quantity: int, required.
- UnitPrice: decimal(18,2), required.
- DiscountPercent: decimal(5,2), optional.
- FinalUnitPrice: decimal(18,2), required.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Index on CartId.
- Index on ProductId.
- Unique index on CartId + ProductId + ProductSizeId + ProductColorId + ProductVariantId.

Relationships:
- Belongs to Carts.
- Belongs to Products.
- Optionally belongs to ProductSizes, ProductColors, ProductVariants.

Validation:
- Quantity must be greater than 0.
- Quantity cannot exceed available selected size/variant stock.
- Required product color/size must be selected before adding to cart.


Table: Orders
Purpose:
Stores order header, customer snapshot, delivery address snapshot, payment, totals, and current status.

Fields:
- Id: uuid, required, primary key.
- OrderNumber: nvarchar(50), required.
- UserId: uuid, nullable, foreign key to Users.Id. Nullable after account deletion.
- CustomerDeleted: bit/boolean, required, default false.
- CustomerNameSnapshot: nvarchar(200), required.
- CustomerPhoneSnapshot: nvarchar(40), nullable/anonymized on account deletion.
- CustomerEmailSnapshot: nvarchar(320), nullable/anonymized on account deletion.
- City: nvarchar(120), required.
- Area: nvarchar(120), required.
- Street: nvarchar(200), required.
- Building: nvarchar(100), optional.
- Notes: nvarchar(1000), optional.
- Subtotal: decimal(18,2), required.
- DiscountTotal: decimal(18,2), required, default 0.
- DeliveryFee: decimal(18,2), required, default 0.
- Total: decimal(18,2), required.
- PaymentMethod: enum PaymentMethod, required.
- PaymentStatus: enum PaymentStatus, required.
- Status: enum OrderStatus, required.
- CreatedAt: datetime required.
- UpdatedAt: datetime required.

Indexes:
- Unique index on OrderNumber.
- Index on UserId.
- Index on Status.
- Index on CreatedAt.
- Index on PaymentStatus.

Relationships:
- Belongs to Users, nullable.
- Has many OrderItems.
- Has many OrderStatusHistory rows.
- Can have AdminNotifications.

Notes:
- Orders must not be deleted when a customer deletes the account.
- Personal identifiers must be anonymized on account deletion.
- Orders start with Status = UnderReview.
- PaymentMethod is CashOnDelivery only.
- PaymentStatus starts PendingOnDelivery.

Fields:
- Id: uuid, required, primary key.
- OrderId: uuid, required, foreign key to Orders.Id.

- CategoryNameSnapshot: nvarchar(200), optional.
- ProductSizeLabelSnapshot: nvarchar(80), optional.

- Quantity: int, required.
- UnitPrice: decimal(18,2), required.
- DiscountPercent: decimal(5,2), optional.
- FinalUnitPrice: decimal(18,2), required.
- LineTotal: decimal(18,2), required.

Indexes:
- Index on OrderId.
- Index on ProductId.

Relationships:
- Belongs to Orders.
- Optionally references Products.

Notes:
- ProductId is nullable so deleted products do not break historical order details.

Indexes:

Rules:

- Percentage must be between 0 and 100.

4. BUSINESS RULES
-----------------

Arabic-only frontend:
- All server-provided display labels/content should support Arabic.
- The frontend remains RTL.
- Keep numbers as English digits. Formatting can be handled by frontend using en-US number formatting.

Admin cannot shop:
- Admin must not add to cart, wishlist, checkout, or place orders.
- Backend must reject cart/wishlist/order mutation endpoints for Admin role.
- Admin can preview storefront products, but shopping actions are not allowed.

Guest cannot add to cart:
- Guests can browse products and categories.
- Guests must login/register as Customer before cart, wishlist, checkout, or order creation mutations.

Customer data isolation:
- Customers can only read/update their own profile, addresses, cart, wishlist, and orders.
- Admin can read all orders and manage store data.
- Never let one customer access another customer's orders.

Account deletion:
- Customer can delete account.
- User is marked IsDeleted = true or anonymized.
- Email and phone are removed/anonymized.
- CustomerProfiles, CustomerAddresses, Cart, CartItems, Wishlist, WishlistItems are deleted.
- Login sessions/tokens are revoked.
- Orders are not deleted.
- Related Orders must set CustomerDeleted = true.
- Related Orders must anonymize CustomerEmailSnapshot and CustomerPhoneSnapshot.
- Admin can still see order number, items, totals, statuses, payment, and history.

Product size pricing:
- Product has BasePrice.
- ProductSizes.Price can override BasePrice.
- ProductVariants.Price can override both ProductSizes.Price and BasePrice if variants are used.
- Cart and orders must store the selected final price snapshot.
- Quantity cannot exceed selected size/variant stock.

Discount priority:
- Product discount has highest priority.
- Brand discount comes next.
- Global discount comes last.
- Discount applies to selected size/variant price.
- Admin must not manually update every product for global discounts.

Product rating:
- Admin cannot create/edit Rating or ReviewCount fields.
- Rating is calculated from approved ProductReviews.
- AverageRating = sum(Rating) / count(approved reviews).
- ReviewCount = count(approved reviews).
- If no reviews, frontend shows no reviews yet.

Cash on delivery:
- Payment method is CashOnDelivery only.
- No card, PayPal, Stripe, or electronic payment fields.
- New order PaymentStatus = PendingOnDelivery.
- When order is Received, PaymentStatus can become PaidOnDelivery.

Order lifecycle:
- New orders start UnderReview.
- Admin can approve or reject UnderReview orders.
- Approved orders can move to Preparing, Shipping, OutForDelivery, Received.
- Every status change appends OrderStatusHistory.
- Customer sees current status and timeline.

Notifications:
- New order creates AdminNotifications row Type = NewOrder.
- Notifications are in-system only.
- Admin can mark one notification as read or all as read.


5. REQUIRED BACKEND APIS
------------------------

General response shape:
Success:
{
  "success": true,
  "data": {},
  "message": "optional Arabic message"
}

Error:
{
  "success": false,
  "error": {
    "code": "validation_error",
    "message": "Arabic error message",
    "fields": {}
  }
}


AUTH APIS
---------

POST /api/auth/login
Purpose:
Login admin or customer.
Access:
Public.
Body:
- email: string, required.
- password: string, required.
Response:
- user: { id, fullName, email, phoneNumber, role }
- token or secure session cookie.
Validation:
- User must exist.
- Password must match.
- IsDeleted must be false.

POST /api/auth/register-customer
Purpose:
Register a new customer account.
Access:
Public.
Body:
- fullName: string, required.
- email: string, required, unique among active users.
- phoneNumber: string, required.
- password: string, required.
Response:
- user profile and auth token/session.
Validation:
- Role is always Customer.
- Email format valid.
- Password meets policy.
- Deleted account email can be reused only if old email was anonymized.

POST /api/auth/logout
Purpose:
End current session.
Access:
Authenticated Admin or Customer.
Body:
- none.
Response:
- success message.
Validation:
- Revoke token/session if server-side sessions are used.

GET /api/auth/me
Purpose:
Return current authenticated user.
Access:
Authenticated Admin or Customer.
Response:
- user: { id, fullName, email, phoneNumber, role }
Validation:
- IsDeleted must be false.

DELETE /api/account
Purpose:
Delete/deactivate current customer account.
Access:
Customer only.
Body:
- password: string optional if requiring confirmation.
Response:
- success message.
Validation:
- Admin cannot call this endpoint as a shopping account.
- Orders remain.
- Personal data is anonymized.
- Cart/wishlist/profile/addresses are removed.


CUSTOMER ACCOUNT APIS
---------------------

GET /api/account/profile
Purpose:
Read current customer profile.
Access:
Customer only.
Response:
- fullName, email, phoneNumber, preferredName.

PUT /api/account/profile
Purpose:
Update profile.
Access:
Customer only.
Body:
- fullName: string, required.
- phoneNumber: string, required.
- preferredName: string optional.
Response:
- updated profile.
Validation:
- Customer can update only own profile.
- Email change should be a separate verified flow if needed.

GET /api/account/addresses
Purpose:
List customer addresses.
Access:
Customer only.
Response:
- array of addresses.

POST /api/account/addresses
Purpose:
Create address.
Access:
Customer only.
Body:
- city, area, street: required.
- building, notes: optional.
- isDefault: optional.
Validation:
- If first address, make default.
- If isDefault true, clear other defaults.

PUT /api/account/addresses/{id}
Purpose:
Update address.
Access:
Customer only.
Validation:
- Address must belong to current customer.

DELETE /api/account/addresses/{id}
Purpose:
Delete address.
Access:
Customer only.
Validation:
- Address must belong to current customer.
- If default deleted, another address may become default.

PUT /api/account/addresses/{id}/default
Purpose:
Set address as default.
Access:
Customer only.
Validation:
- Address must belong to current customer.


PRODUCT APIS
------------

GET /api/products
Purpose:
List active storefront products.
Access:
Public.
Query:
- page, pageSize.
- categorySlug/categoryId optional.
- brandId optional.
- minPrice/maxPrice optional.
- sort optional.
Response:
- products with images, brand, category, colors, sizes, calculated rating, and effective discount.
Validation:
- Only active products for public.

GET /api/products/{id}
Purpose:
Get product by id.
Access:

6. API NOTES BY GROUP
---------------------

Auth:
- Public login/register endpoints return authenticated session data.
- Authenticated endpoints must reject deleted users.
- Registration is customer-only. Admin creation should be handled separately by seed/admin tooling.

Customer:
- Customer endpoints must use current authenticated user id, not user id from request body.
- Account deletion must be transactional:
  1. Mark user deleted/anonymize user.
  2. Delete profile/addresses/cart/wishlist.
  3. Anonymize order personal identifiers.
  4. Revoke sessions.

Products:
- Public products include calculated rating and active discount summary.
- Admin product create/update must support colors, sizes, and optional variants.
- Do not accept manual Rating or ReviewCount.
- Size prices and variant prices must be decimal values.

Categories and brands:
- Public lists show active rows only.
- Admin can create/update/deactivate.
- Slugs should be unique and stable.

Cart:
- Cart price should be recalculated on add/update using selected size/variant and current discount rules.
- Store UnitPrice and FinalUnitPrice snapshots in CartItems.
- Cart totals are derived from FinalUnitPrice * Quantity.

Wishlist:
- Wishlist is per customer.
- Admin and guest mutations forbidden.

Orders:
- Order creation copies cart item snapshots into OrderItems.
- Order totals must use cart/order final unit prices.
- Order customer and address snapshots must stay with the order.
- Admin status changes must be immediately visible to customer order views.

Reviews:
- Product rating is a query/projection from ProductReviews.
- If moderation is enabled, only approved reviews count.

Settings:
- Store settings are admin-only except public announcement.
- Frontend can cache announcement but should refresh after admin updates.

Discounts:
- Effective discount resolver:
  1. Check active Product discount by ProductId.
  2. Else check active Brand discount by BrandId.
  3. Else check active Global discount.
  4. Else no discount.
- Active means IsEnabled = true and date range contains current time.

Notifications:
- New order creation writes AdminNotifications row.
- Read state persists per notification.
- Clicking notification should open the related order details.
